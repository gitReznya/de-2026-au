# Промт для адаптации команд под другой вариант

```text
Ты помощник по подготовке к ДЭ 09.02.06.

Есть готовые команды для варианта:
- domain: au-team.irpo
- VLAN HQ-SRV: 120
- VLAN HQ-CLI: 220
- VLAN MGMT: 888
- HQ-SRV: 192.168.120.0/27
- HQ-CLI: 192.168.220.0/27
- MGMT: 192.168.88.0/29
- BR-SRV: 192.168.30.0/28
- GRE: 10.10.10.0/30

Нужно заменить параметры под новый вариант:
- новый domain: <...>
- новый VLAN HQ-SRV: <...>
- новый VLAN HQ-CLI: <...>
- новый VLAN MGMT: <...>
- новая сеть HQ-SRV: <...>
- новая сеть HQ-CLI: <...>
- новая сеть MGMT: <...>
- новая сеть BR-SRV: <...>
- новый SSH port: <...>
- новый пользователь: <...>
- новый пароль: <...>

Задача:
1. Аккуратно заменить IP, VLAN, domain, DNS-записи, DHCP range, OSPF network, NAT source, NFS allow, nginx server_name.
2. Вывести таблицу старое → новое.
3. Вывести потенциально опасные места, где можно ошибиться.
4. Не менять смысл команд.
5. Если не уверен — пометить как проверить.

Вот команды:
<ВСТАВЬ_КОМАНДЫ>
```
